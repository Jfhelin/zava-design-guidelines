# Zava UI Patterns

## Preferred Patterns

- header with logo, page title, and one primary action
- card-based content grouping
- form-first workflows with explicit labels
- status pills and progress summaries
- clear section spacing
- responsive two-column layouts that collapse cleanly to one column

## Component Guidance

- buttons should be obvious and sparing
- tables should be readable and lightly bordered
- cards should carry one idea each
- forms should avoid unnecessary fields
- empty states should be practical, not promotional
- sign-in screens should feel clean and reassuring rather than corporate-heavy
- borders and dividers can be more important than fills
- inputs and selects can follow Bootstrap-like default geometry
- primary header buttons can use a teal gradient pill treatment when appropriate

## Layout Guidance

- use a consistent page width
- give actions predictable placement
- avoid hiding core actions in menus
- keep important context above the fold where possible
- prefer rounded containers, buttons, and content shells
- allow the logo to be the main brand signal instead of forcing color into the UI
- keep the footer information-dense and clearly columnized
